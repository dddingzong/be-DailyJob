# be-DailyJob

> 오전: step5 쿠키를 이용한 로그인 기능 구현
>
> 오후: Step6 동적인 HTML 구현

### 1. 오늘 완료한 것
- 오늘 실제로 끝낸 기능/수정/학습:
  - 세션 기반 동적 헤더 구현 
  - 페이지별 접근 제어 구현 
  - TemplateEngine 단위 테스트 3개, ViewController 통합 테스트 11개 작성

### 2. 오늘 가장 의미있었던 기술적 판단
- 내가 내린 판단: HTML을 Java에 하드코딩하지 않고 fragment 파일로 분리
- 그렇게 판단한 이유: Java 안에 HTML을 직접 조립하면 디자인 수정 시 Java 코드를 건드려야 함
- 이 판단이 코드 품질/테스트/유지보수에 미친 영향: Controller는 데이터 결정, TemplateEngine은 치환만 담당. fragment 추가만으로 확장 가능

### 3. 가장 막혔던 문제와 해결 과정
- 문제 상황 또는 증상: 정적 파일인 index.html을 세션에 따라 다르게 응답해야 하는 상황
- 처음 추측한 원인: 정적 파일 서빙 구조를 전면 수정해야 한다고 생각
- 실제 원인: 조건이 붙으면 동적 처리이므로 해당 경로만 RouteConfig에 등록하면 됨
- 해결 방법: `GET /`을 RouteConfig에 등록하여 ViewController가 처리. CSS/JS/이미지는 기존 정적 서빙 유지
