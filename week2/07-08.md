### 2. 가장 막혔던 문제와 해결 과정
- 문제 상황 또는 증상: 302 redirect 응답에서 Set-Cookie가 동작하는 원리가 이해되지 않았음. redirect인데 브라우저가 쿠키를 저장하는 시점이 언제인지 혼란
- 처음 추측한 원인: redirect면 응답 본문이 없으니 쿠키도 무시될 수 있다고 생각
- 원인을 확인하기 위해 시도한 방법: HTTP 스펙에서 Set-Cookie 헤더의 처리 시점을 확인
- 실제 원인: 브라우저는 302 응답의 헤더를 먼저 처리한 후 Location으로 리다이렉트함. 상태 코드와 관계없이 응답 헤더의 Set-Cookie는 항상 처리됨
- 해결 방법: `HttpResponse.redirect(location, cookie)` 오버로딩으로 302 + Set-Cookie + Location을 한 응답에 담아 전송
